# Project Workflow

All contributors and AI agents must adhere to the following workflow when executing tasks in this project.

1. **Review Documentation:**
   Begin every task by reading and reviewing all documents located in the `docs/` folder to understand the current architecture, guidelines, and context.

2. **Design Matching:**
   Verify that the current task matches up with an existing design document found in the `docs/` folder.

3. **Design First (If Necessary):**
   If no existing design covers the current task, you must:
   - Write a new design document for the task.
   - Save the design document in the `docs/` folder.
   - Commit the new design document to the repository **before** beginning any implementation work.

4. **Implementation:**
   Proceed with the implementation only after ensuring a matching design document exists and is committed.

5. **Completion:**
   Ensure all changes are checked in, pass all relevant tests, and are pushed to GitHub. 
   If pushing a newly created branch for the first time, create a pull request for that branch before declaring "Job's done!".
