# Documentation Structure

Welcome to the project documentation! To keep our documentation easy to find and maintain, we use the following rational structure:

- **`hardware/`**: Contains documents related to the physical hardware components, including ESP32 devices, architecture, provisioning workflows, and testing strategies.
- **`ux/`**: Contains User Experience (UX) specifications, phone layout details, PWA specs, app icon designs, mockups, and screenshots.
- **`archive/`**: Contains historical, deprecated, or auto-generated design documents that are no longer actively maintained but kept for reference.

## Guiding Principles

- Keep related assets (like images or screenshots) within a subfolder close to the documents that reference them.
- Maintain a clean root directory (`docs/`) to avoid clutter.
- Update this `README.md` if introducing a new top-level category.
